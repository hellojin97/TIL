# MySQL

## 데이터 베이스 모델링(설계)

- 개념적으로 어느 정도 파악 후 추후에 공부 필요
- 데이터 베이스 모델링은 설계도를 그리는 과정.
- 예를 들어 건축물을 짓기 위해서는 설계도를 그려야하는 것처럼 데이터베이스를 만들기 위해서는 모델링이 필요하다.
  - 데이터 베이스 모델링은 **프로젝트 설계 부분**에서 중요한 부분.

### 프로젝트 진행 단계

- 프로젝트 : 현실세계에서 일어나는 업무를 컴퓨터 시스템으로 옮겨놓는 과정. 그에 대한 결과가 소프트웨어를 만드는 전체 과정이라고 생각하면 됨.
- 즉, **현실 세계**에서 일어나는 업무를 **컴퓨터 시스템**으로 옮긴 것이 ***소프트웨어***, 그 소프트웨어를 만들기 위한 과정이 ***프로젝트***

- 프로젝트 모델, 폭포수 모델
  - **폭포수가** 떨어지듯이 작업을 하는 것. 즉, **단계별 순차적 프로젝트** 진행 모델이라는 의미
  - 단계별 프로젝트
    1. 프로젝트 계획
    2. 업무 분석
    3. 시스템 설계
    4. 프로그램 구현
    5. 테스트
    6. 유지보수

### 데이터 베이스 모델링

- 일반적으로 **시스템 설게 부분**에 들어감.
- 우리가 살고 있는 세상 **(현실세계)** 을 **실제 데이터베이스** 안으로 넣기 위한 작업

  ```text
  현실세계) 슈퍼마켓
  직원 : 실제 사람
  고객 : 물건을 구매, 물건을 가져감.
  진열된 물건 : 실제 물건
  ⬇️
  데이터베이스) 슈퍼마켓 DB
  직원 테이블 : 직원들의 정보
  회원 테이블 : 회원 들의 정보
  구매 테이블 : 회원 구매로 인한 구매 내역 정보
  물건 테이블 : 물건들의 대한 정보
  ```

- 실제 물건이나 사람을 넣는것이 아닌 그에 대한 정보를 저장하는 것이 데이터베이스 모델링.
  - 즉, ***현실세계에서 얻은 정보를 토대로 데이터베이스 안에 넣기 위해 테이블을 결정***하는 과정.
  - Tips
    - 데이터베이스 모델링은 정답이 없다. 일반적인 모범답안 정도는 있다.
    - 좋은 모델링과 나쁜 모델링은 분명히 존재한다.
    - 건축물 설계 단계 또한 비슷하다.

### 쇼핑몰 데이터 베이스를 통해 전체 구성도 알아보기

- **열**과 **열 이름**
  - 열은 테이블의 새로 부분.
  - 열 이름은 각 열을 구분하기 위한 이름, 각 테이블 내에서는 서로 이름이 달라야 함.
- **행**  
  - 실질적인 **진짜 데이터**, 행 데이터 라고도 부름. ***행의 개수가 즉, 데이터의 개수***.

- **기본 키** `Primary Key`
  - 테이블 내에서 각각의 행을 구분하는 유일한 값 즉, 각 행의 id
  - 중복 허용 안됨.  테이블마다 각 하나만 지정 가능(약속).
  - 네이버 회원이면서 id 가 없는 사람이 없듯이 필수적이어야 함.
- **데이터 형식**
  - 아이디는 문자, 가격은 숫자, 등등 열마다 데이터 형식을 어떻게 할 지 선결정 하는 것이 데이터 형식.
