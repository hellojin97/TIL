# DB, Database 란 무엇인가

해당 내용은 아래에 해당하는 내용으로 구성되었습니다.

- DB 의 기본개념
- DB 는 왜 등장했나
- DB 의 구조

## DB 의 기본개념

- 사전적 정의 : **정보처리 시스템** 중 하나로 데이터 수집, 저장, 생성 및 분배 역할
  - 정보처리 시스템이란?
    - 데이터를 가공해서 정보로 바꿔줌
- 즉, 세상의 다양한 데이터를 효과적으로 저장, 운영하기 위한 기술.

## DB 는 왜 등장했나

- 파일 처리 시스템의 한계를 극복하기 위해 등장
  - 파일 처리 시스템이란?
    응용프로그램(Application)마다 데이터 파일이 종속적인 성격과 중복성을 갖게 되는데 데이터, 응용프로그램 서로의 입장에서 불편한 내용인데 이유는 아래와 같다.
    - 종속 : 데이터 파일이 바뀔때마다 어플리케이션에 영향을 주게 됨.
    - 중복 : 공간의 낭비 발생.
  - DB 는 종속성과 중복 문제를 해결하기 위해 등장.

- 기존의 파일 처리 시스템

```text
  Data File 1 -> Application 1
  Data File 2 -> Application 2
  Data File 3 -> Application 3
```

- DB 의 소통 구조

```text
Database -> DBMS -> App1, App2, App3
```

- 작동 방식

  - 데이터 베이스는 저수지처럼 데이터를 담아둠
  - 각각 응용 프로그램은 DBMS 를 통해 원하는 데이터를 가져올 수 있음.
- 개선된 요소
  1. 프로그램과 데이터를 분리(종속성 해결)
  2. 데이터 추상화
  3. 다중 뷰(다양한 사용자 대응)
  4. 동시 접근, 동시 제어 가능

- 단점
  - 운영비 증대
  - 백업이 가능하나 부하가 발생
  - 자료를 한 곳에 몰아넣다보니 복잡

## DB 의 구조

### 3단계구조 (ANSI/SPARC 구조)

**스키마** - DB 의 구조와 데이터 간의 관계를 표현한것.

- 외부 단계
  - 사용자가 접근하는 영역
  - 외부 스키마 : 외부 사용자를 위한 구조
- 개념 단계
  - 관리 조직이 DB 전체를 추상화
  - 왜 추상화를 할까?  
    조금 더 접근하기 쉽게 전체 데이터가 어떻게 형성되어 있는 지 판단하기 위함
  - 개념 스키마 : 전체적이고 통합된 DB 구조, 조직에서 관리
- 내부 단계
  - 저장 DB에 어떻게 저장 시킬지(물리적)
  - 내부 스키마 : 물리적인 저장 데이터 구조
