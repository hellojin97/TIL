### 왜 Hadoop 인가?

![더그 커팅과 하둡](https://t1.daumcdn.net/cfile/tistory/997CA3505B3880CD02)

하둡은 GFS(Google File System)에서 착안하여 야후의 더그 커팅과 그의 동료가 만들어 냈음.  
당시 더그 커팅의 아이가 가지고 놀던 장난감이 노란색 코끼리 인형이었고 이름이 하둡(Hadoop)이었음.  
그래서 하둡(Hadoop)임.

### Hadoop Ecosystem

![Hadoop Ecosystem](https://user-images.githubusercontent.com/60086878/111483142-de938580-8777-11eb-92c6-7527eeb4fdeb.png)

> **대규모 데이터 처리를 위한 오픈소스 플랫폼으로, 하둡의 주요 구성 요소와 함께 다양한 프로젝트, 라이브러리 및 도구로 구성되어 있음.**

#### 2가지 유형

##### Hadoop Core Components

- HDFS, MapReduce 가 Hadoop 의 주요 구성요소.
  - HDFS : 빅데이터를 저장하기 위한 분산 파일 시스템
  - MapReduce : 분산 환경에서 빅데이터 처리를 위한 프로그래밍 모델.

##### Hadoop Ecosystem Projects

- 다양한 유형의 데이터를 처리하기 위한 라이브러리 및 도구를 제공
  - Apache Pig
  - Apache Hive
  - Apache HBase
  - APache Spark
  - APache Storm
  - Apache Kafka
  - APache Flume
