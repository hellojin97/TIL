### Hadoop Ecosystem

![Hadoop Ecosystem](https://user-images.githubusercontent.com/60086878/111483142-de938580-8777-11eb-92c6-7527eeb4fdeb.png)

> Hadoop 을 기반으로 하는 여러 오픈 소스 및 프로젝트 도구의 집합체를 의미. 이를 활용하면 대규모 데이터 처리, 분석, 시각화, 저장 등의 작업을 수행할 수 있음.

- core hadoop ecosystem

#### Hadoop Distributed File System(HDFS)

```text
HDFS {
  클라이언트 --> 네임노드
  네임노드 --> 데이터노드1
  네임노드 --> 데이터노드2
  네임노드 --> 데이터노드3
  데이터노드1 -->|데이터 블록 복제| 데이터노드2
  데이터노드1 -->|데이터 블록 복제| 데이터노드3
  데이터노드2 -->|데이터 블록 복제| 데이터노드1
  데이터노드2 -->|데이터 블록 복제| 데이터노드3
  데이터노드3 -->|데이터 블록 복제| 데이터노드1
  데이터노드3 -->|데이터 블록 복제| 데이터노드2
}
```

- 분산 저장 시스템
- 대용량 데이터를여러 대의 컴퓨터에 분산하여 저장하는 분산 파일 시스템.
- Hadoop 버전의 GFS, 클러스터의 컴퓨터들에 분산 저장하는 시스템.
- 데이터의 여분 복사본까지 만들어 컴퓨터가 손상되면 백업 복사본을 사용해 자동으로 손실을 회복함.

#### Yarn

```text
YARN {
  클라이언트 --> ResourceManager
  ResourceManager -->|요청 받음| NodeManager1
  ResourceManager -->|요청 받음| NodeManager2
  ResourceManager -->|요청 받음| NodeManager3
  NodeManager1 -->|리소스 할당| 애플리케이션1
  NodeManager1 -->|리소스 할당| 애플리케이션2
  NodeManager2 -->|리소스 할당| 애플리케이션3
  NodeManager2 -->|리소스 할당| 애플리케이션4
  NodeManager3 -->|리소스 할당| 애플리케이션5
}
```

- 데이터 처리 부분을 담당
- 컴퓨터 클러스터의 리소스를 관리
  - 각 노드의 리소스(CPU, 메모리 등)를 다양한 어플리케이션들이 효율적으로 공유하여 사용할 수 있도록 함.
  - 각 어플리케이션이 필요로 하는 리소스를 동적으로 할당, 어플리케이션 간의 리소스 간섭을 방지함.
- Hadoop 2.0 부터 도입되었으며, 이전 버전에서는 MapReduce 가 리소스 관리를 담당했음.
- 어플리케이션마다 사용할 수 있는 자원의 종류와 크기를 구분
  - 두 가지 유형의 자원을 지원.
    1. CPU 와 같은 계산 자원
    2. 메모리와 같은 스토리지 자원
- Yarn 은 대규모 분산 데이터 처리 시스템에서 사용되며 MapReduce 를 비롯한 어플리케이션들이 Yarn 기반으로 실행됨.

##### Yarn 의 구성 요소

```text
+-------------------------------------------+
|                   Client                  |
+-------------------------------------------+
        |
        |   Submit application request
        v
+-------------------------------------------+
|                 ResourceManager           |
+-------------------------------------------+
        |                            |
        |                            |
        |   Allocate resources      |   Track node health
        |                            |
        v                            v
+------------------------+      +------------------------+
|      Application       |      |      Application       |
|       Master 1         |      |       Master 2         |
+------------------------+      +------------------------+
        |                            |
        |                            |
        |   Execute tasks           |   Execute tasks
        |                            |
        v                            v
+------------------------+      +------------------------+
|      Node Manager 1     |      |      Node Manager 2     |
+------------------------+      +------------------------+
        |
        |
        |   Monitor container status
        v
+------------------------+
|        Container       |
+------------------------+
```

1. ResourceManager

   - 클러스터의 전반적인 자원관리를 담당
   - 클러스터의 모든 노드에 대한 리소스 정보를 유지하고, 각 노드의 상태를 추적
   - 클러스터에서 사용 가능한 자원에 대한 정보를 모니터링하고, 어떤 어플리케이션이 자원을 요청하는지 추적함.

2. ApplicationManager

##### Yarn 의 두 가지 자원 관리

1. 노드(Node)

   - 클러스터 내의 물리적인 서버 또는 가상 머신
   - CPU, 메모리, 디스크 네트워크 등의 자원을 가지고 잇음
   - Yarn 은 클러스터 내의 모든 노드를 관리, 각 노드의 자원 상태를 모니터링하고 필요에 따라 자원을 할당 및 해제함.

2. 컨테이너(Container)

   - 각 분산 작업을 실행하는 데 필요한 자원을 캡슐화하는 단위
   - 각 컨테이너는 특정 노드에서 실행, 해당 노드의 자원을 이용하여 작업을 처리함.
   - Yarn 은 각 작업에 필요한 자원을 계산 -> 컨테이너 생성하여 자원을 할당
