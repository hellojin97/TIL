# 점근적 표기법, Asymptotic Notation

> ***계수와 나머지 항목을 제거하고 최고 차수의 N을 표현하는 것이 점근적 표기법이다.***

## 알고리즘의 중요성

- 알고리즘의 실행 시간은 컴퓨터가 알고리즘 코드를 실행하는 속도에 의존
- 속도는 컴퓨터의 처리 속도, 사용된 언어 종류, 프로그래밍 언어를 컴퓨터가 실행할 수 있는 코드로 바꾸는 컴파일러의 속도 등에 달려 있음.
- 알고리즘의 실행 시간의 두가지 분류
  - 입력값의 크기에 따른 알고리즘의 실행 시간
  - 입력값의 크기에 따른 함수의 성장률, 실행 시간의 성장률(rate of growth)

## 성장률(Rate of Growth)

```text
f(n) = 6n^2 + 100n + 300
```

- 위와 같은 식에서 `n` 값이 어느 정도 커지면 `6n^2` 이 초기에 더 컸던 나머지 항목 `100n + 300` 보다 더 크다는 것을 알 수 있다.

     ![Alt text](https://cdn.kastatic.org/ka-perseus-images/0642ea78ce621e53dbe7f45881a97786c7262635.png)

- 계수 6과 나머지 항목을 제거한다 가정한다면 이 알고리즘의 실행 시간은 `n^2` 인 ***기하급수적인 증가량***을 확인 할 수 있다. **(점근적 표기법은 계수는 중요하지 않음.)**

```text
f(n) = 0.6n^2 + 1000n + 3000
```

- 위와 같은 식도 처음엔 나머지 항목이 더 큰값으로 표시되겠으나 결국엔 **교차점은 반드시 존재하여 `n^2` 이 크게 됨.**

     ![Alt text](https://cdn.kastatic.org/ka-perseus-images/d2d40c938c1bab9f413c83164fec8ae9945e402b.png)

- 점근적 표기법은 **중요하지 않은 항과 계수를 제거하여 이해를 방해하는 불필요한 부분을 없애서 알고리즘의 실행 시간에서 중요한 부분인 성장률에 집중하는 것.**
